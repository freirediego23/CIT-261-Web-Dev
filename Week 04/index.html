Object Methods

Methods are actions that can be applied to object properties in JS.

Object.entries

Turns objects properties into one array

var id = {name: "Diego", age:20}
var info = Object.entries(id);

Object.keys

This method picks only the key values of an object and returns an array.

var city = {name: "Quito", population:3 million}
var info = object.keys(city);

Object.values

This method will only return the values of object properties

var cars = {name: "Toyota", number:5}
var info = Object.values(cars);

Object.freeze

This method will prevent changes to be applied to your object.
I think that's why it is called freeze.

var id = {name: "Diego", age:20};
var freeze = Object.freeze(id);
freeze.name = "Fernando";
freeze.age = 30;

Object Instantiation

Functional Instantiation

Functional instantiation defines all methods in the constructor function.
This is considered good when you are not working with a lot of instances.
New sets of methods are creating for every instance and it may consume memory.

Object Functional-shared

Instead of declaring all methods in each function, we now store every method
in an object. The object sotring our methods will be referenced in any new object we want.

Object Prototype

Prototype has a similar behavior with function-shared because both
use an object to store methods. We use Object.create and reference the object
that contains the method as our prototype for the new object created.

Object Pseudoclassical

This method simplifies typing. We don't store our methods in an object
to be later called. We use the word "this" to replace Object.create
and we add our methods to the prototype
